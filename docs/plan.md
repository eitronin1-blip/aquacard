# План: фикс импорта профиля + «синхронизация карт» AquaDX — PENDING APPROVAL

Статус: **ОЖИДАЕТ ОДОБРЕНИЯ** (ralplan, без --interactive). Изменения НЕ внесены.

## RALPLAN-DR summary

**Принципы**
1. Чинить корень, а не симптом; не маскировать ошибки парсинга «сетевыми».
2. Не обещать невозможного: ограничения платформы (HCE-F `02FE`) и API (нет списка карт) — честно в UI.
3. Минимальная инвазивность: точечные правки, без переписывания модулей.
4. Диагностируемость: ошибки должны различать «сеть / HTTP / парсинг».

**Decision drivers (топ-3)**
1. Импорт профиля сейчас всегда падает на валидном ответе (баг типа `regTime`).
2. У пользователя уже есть карты в AquaDX — он хочет их «подтянуть».
3. Технические факты: (а) у AquaDX НЕТ эндпоинта списка карт пользователя; (б) физические карты имеют IDm `012E`, а Android HCE-F вынуждает `02FE` → их нельзя эмулировать телефоном.

**Viable options для «синхронизации карт»**
- **A. Импорт/восстановление карты по access code (вручную)** — пользователь вставляет 20-значный код, приложение восстанавливает IDm и сохраняет; помечает «эмулируемая» только если IDm начинается с `02FE`.
  - + Реально работает без приватного API; покрывает восстановление созданных в приложении карт.
  - − Не «авто-синк»; физические карты сохраняются как справочные (не эмулируемые).
- **B. Просмотр профилей аккаунта по username** (расширить текущую вкладку, `card/user-games`).
  - + Публично, без авторизации; показывает все игры разом.
  - − Это статистика, не карты; не даёт IDm/access code.
- **C. Ничего (оставить только профиль по username).**
  - + Ноль работы. − Не решает запрос пользователя.

Рекомендация: **A + B**, с явным сообщением об ограничении HCE-F. (Авто-синк карт из AquaDX отклонён: невозможен — нет API + нельзя эмулировать `012E`.)

## Часть 1 — Фикс импорта профиля (баг, высокий приоритет)

**Корень:** `data/AquaApi.kt` → `AquaUser.regTime: String? = null`, но API отдаёт число (`"regTime":1748046905871`). kotlinx.serialization бросает `SerializationException`; в `ProfileTab` (MainAppScreen.kt:745) ловится в общий `catch` и показывается «Не удалось загрузить… Проверьте URL» — вводит в заблуждение (сеть исправна).

**Изменения:**
1. `AquaUser.regTime` → `Long? = null`. Свериться с живым JSON по всем полям модели (`name/serverRank/rating/plays/aquaUser{username,displayName,country,regTime,profileBio}`) — единственное несоответствие типа именно `regTime`.
2. `AquaApi`: в `Json{}` добавить `isLenient = true` (к уже включённым `ignoreUnknownKeys`, `coerceInputValues`) — терпимость к мелким отклонениям сервера.
3. `ProfileTab` обработка ошибок: различать ветки — `HttpException` (показать код/тело), `IOException` (сеть/URL), прочее (парсинг). Чтобы будущие поломки были диагностируемы.
4. (Опц.) В `AquaService` оставить `@GET`; эндпоинт принимает GET — ок.

**Acceptance criteria (тестируемо):**
- Запрос `mai2` для `username=Sigma` (публично доступен) отображает `name`, `rating`, `plays`, `serverRank` без ошибки.
- `chu3` для того же username — тоже успех.
- Несуществующий username — показывается осмысленная ошибка (не краш).
- Юнит-тест десериализации: фикстура реального ответа `user-summary` парсится в `GameSummaryResponse` без исключения (включая `regTime` как число).

## Часть 2 — «Синхронизация карт» (фича, реалистичный объём)

**Жёсткие факты (подтверждены исходниками AquaDX и Android):**
- `CardController` AquaDX: только `summary` (нужен token), `user-games` (public), `link`/`unlink` (token), `default-game` (public). **Эндпоинта «список карт пользователя» нет**; `user.cards` наружу не отдаётся (даже extId публично не утекает).
- Физические Amusement IC карты: IDm с префиксом `012E`. Android HCE-F принудительно требует NFCID2 `02FE…`. ⇒ **физическую карту нельзя эмулировать телефоном** (можно лишь хранить как справочную и смотреть её игровой профиль по username).

**Форма AquaDX «Link Card» (факт из UI):** принимает на выбор ЛИБО **Access Code** (20 цифр, группы по 4: `0008 1234 5678 8765 4321`), ЛИБО **Serial Number** = FeliCa IDm в hex с двоеточиями (`01:2E:1A:2B:3C:4D:5E:6F`). Наши `02FE`-карты привязываются обоими способами. ⇒ задача приложения — **выдать оба значения в точном формате формы и дать вставить их в один тап**.

**Реализация (приоритет = Linking Helper):**
1. **Linking Helper (ядро фичи).** На экране карты для каждой карты показать ДВА готовых к вставке значения с кнопками «копировать», ровно в формате формы AquaDX:
   - **Serial Number (IDm):** `02:FE:00:00:00:00:00:01` — IDm побайтно через `:`, верхний регистр.
   - **Access Code:** `0021 5609 8321 6036 2497` — 20 цифр группами по 4 (значение из `aquaAccessCode(idm)`).
   - Кнопка **«Привязать на AquaDX»**, открывающая страницу Link Card (по настраиваемому base URL → web-фронт), + краткая памятка «вставь Serial Number или Access Code → выбери игры → Link».
2. **Импорт существующей карты (по Serial Number ИЛИ Access Code).** Диалог принимает любой из двух форматов (с пробелами/двоеточиями или без — нормализуем):
   - Serial Number → `idm = нормализованный hex` (16 hex). Это самый прямой путь (Serial = IDm).
   - Access Code → восстановить IDm: `idm = BigInteger(digits).toLong()`, `hex = String.format("%016X", idm)`; round-trip `aquaAccessCode(hex) == digits` для проверки.
   - Сохранить как `Card`; `emulatable = idm.startsWith("02FE")`. Если нет — пометка «Физическая карта · эмуляция недоступна (HCE-F требует 02FE)», кнопка «Эмулировать» скрыта. Привязку на AquaDX это не ломает — Serial/Access всё равно показываем для линка.
3. **Профиль аккаунта (расширить вкладку):** по `username` дёргать `card/user-games`, показывать все игры (mai2/chu3/ongeki/wacca) списком (имя/рейтинг/последний вход).
4. **UI-честность:** пояснение — «AquaDX не отдаёт список карт через API. Эмулируются только карты с IDm `02FE` (созданные в приложении). Физическую карту (`012E`) телефон эмулировать не может, но её Serial/Access всё равно можно привязать и смотреть профиль».

**Модель данных:** в `Card` добавить опционально `emulatable: Boolean = true` (или вычислять из `idm.startsWith("02FE")` без хранения — предпочтительно вычислять, без миграции Room).

**Acceptance criteria:**
- Для карты `02FE000000000001` приложение показывает Serial Number `02:FE:00:00:00:00:00:01` и Access Code `0021 5609 8321 6036 2497`, каждый копируется одной кнопкой; формат совпадает с формой AquaDX.
- Импорт Serial Number `02:FE:00:00:00:00:00:01` (с двоеточиями) и Access Code `00215609832160362497` дают одну и ту же карту `IDm=02FE000000000001`, помеченную «эмулируемая».
- Импорт физической карты (Serial `01:2E:…` / её Access) создаёт карту «не эмулируемая», без кнопки эмуляции, но с показом Serial/Access для линка.
- Вкладка профиля по username показывает все игры из `card/user-games`.
- Нигде не утверждается, что физическую карту можно эмулировать.

## Часть 3 — Дизайн: минималистичный тёмный, индиго-акцент

Цель: убрать дефолтный Material-фиолетовый/бирюзу («вырвиглаз»), сделать спокойный современный тёмный UI с одним сдержанным акцентом и чёткой иерархией.

**Цветовые токены** (Compose `darkColorScheme`, **dynamicColor выключен**):
- `background #0E0F11`, `surface #16181C`, elevated/`surfaceVariant #1E2126`, `surfaceContainerHighest #24272D`
- `outline #2A2E35`, `outlineVariant #232730`
- текст: `onSurface #E6E8EB`, вторичный `onSurfaceVariant #9BA1A8`
- акцент: `primary #7C9CFF`, `onPrimary #0B1020`, `primaryContainer #1B2233`, `onPrimaryContainer #C9D5FF`
- ошибки: `error #FF6B6B`, `errorContainer #2A1A1C`, `onErrorContainer #FFC2C2`
- Запрет dynamic color (Material You) — иначе системная палитра «уезжает» в кислоту.

**Типографика**
- Заголовки экранов `titleLarge`/`headlineSmall`, вес ~600, `#E6E8EB`.
- Вторичное/подписи `bodySmall`/`labelMedium`, `#9BA1A8`.
- Микро-заголовки секций `labelSmall`, UPPERCASE, letterSpacing ~0.08em, `#9BA1A8`.
- Все идентификаторы (IDm, Serial, Access, Konami) — `FontFamily.Monospace`.
- Один системный шрифт + моно; без зоопарка.

**Компоненты**
- Карточки: `surface`, радиус 20dp, граница 1px `outline`, без тяжёлых теней — глубина слоями; отступ 16–20dp.
- Карта в списке = стилизованная «карточка»: тонкая акцентная полоса в цвете карты (приглушённо), имя крупно, IDm моно-серым, статус-чип (Эмулируемая `02FE` / Физическая).
- Кнопки: основное действие — filled индиго (`onPrimary`), радиус 14dp; вторичные — tonal/text; продуманные press/disabled.
- Чипы игр: outlined; выбранный — `primaryContainer`/`onPrimaryContainer`.
- Поля ввода: filled `#1E2126`, без жёстких рамок, фокус — индиго 1.5px; моно для hex/кодов.
- Linking Helper: две моно-строки (Serial / Access) + иконка «копировать», haptics на копирование.
- Навигация/табы: минимальные, индикатор индиго, неактивные `#9BA1A8`.
- Экран эмуляции: почти-чёрный фон, центральное пульсирующее NFC-кольцо индиго (анимация alpha/scale), крупный текст «Приложите к ридеру», снизу имя карты + IDm. Спокойно.
- Пустые состояния: контурная иконка + короткий текст `#9BA1A8`.

**Ритм/пространство:** сетка 4dp; секции через 16–24dp; не одинаковый padding везде. Радиусы: карточки 20, кнопки 14, поля 12, чипы full.

**Реализация (Compose):**
- Новый пакет `ui/theme/`: `Color.kt` (токены), `Type.kt` (Typography), `Theme.kt` — `AquaCardTheme` на `darkColorScheme(...)`, `dynamicColor=false`, always dark; edge-to-edge + светлые иконки статус-бара под тёмный фон.
- `themes.xml`: `Theme.AquaCard` → тёмная NoActionBar (фон `#0E0F11`); остальное темит Compose.
- `MainActivity`: обернуть контент в `AquaCardTheme`.
- Рефактор `MainAppScreen.kt` под токены/компоненты — **одним проходом вместе с Linking-фичей**.

**Acceptance (дизайн):**
- Нигде нет дефолтного фиолетового/бирюзы; единственный акцент — индиго `#7C9CFF`.
- Тёмная тема фиксированная, dynamic color выкл (на «цветных» прошивках не уезжает).
- Контраст текста ≥ WCAG AA; идентификаторы моноширинные.
- Единый визуальный язык на экранах: список / карта (+Linking Helper) / эмуляция / профиль / настройки.

## Architect — антитезис и компромиссы

- **Сильнейший контр-довод против Option A:** «восстановление по access code» бесполезно, если у пользователя физические карты — он получит лишь нередактируемые справочные записи. **Ответ/синтез:** да, и это честно; ценность A — восстановление *созданных в приложении* карт и каталогизация физических с показом их профиля; ожидания корректируются п.3 (UI-честность). Лучше дать правду, чем «магический синк», который не работает.
- **Tradeoff:** хранить `emulatable` как поле (миграция Room) vs вычислять из `idm`. Выбор: **вычислять** (нет миграции, один источник истины). Минус — нельзя пометить «физическую 02FE» вручную (краевой случай ~нулевой).
- **Tradeoff:** `isLenient` маскирует часть проблем сервера. Принимаем — выгода устойчивости > риск, т.к. парсим в строгие типы и есть юнит-тест-фикстура.

## Critic — проверки качества
- Принципы↔опции согласованы (честность UI ↔ Option A/B; корень ↔ фикс типа).
- Альтернативы рассмотрены честно (C — «ничего» — явно отклонён).
- Риски и митигации заданы (см. ниже).
- Acceptance criteria тестируемы (конкретные username/коды/ожидания).
- Шаги верификации конкретны (юнит-тест фикстуры + ручная проверка на устройстве).

**Риски / митигации**
- R1: после фикса всплывёт другой тип-mismatch на иных полях. → Митигация: юнит-тест на реальной фикстуре + `isLenient`.
- R2: пользователь решит, что физическую карту всё же можно эмулировать. → Митигация: явные плашки и скрытие кнопки эмуляции для не-`02FE`.
- R3: восстановление IDm неоднозначно для high-bit (Konami magnetic). → Митигация: round-trip-валидация при импорте, предупреждение.

## ADR
- **Decision:** (1) Исправить тип `regTime` (+ устойчивость парсинга и диагностику ошибок). (2) Вместо невозможного авто-синка карт — импорт по access code + расширенный просмотр профилей по username, с честным UI об ограничениях.
- **Drivers:** баг ломает импорт на валидном ответе; нет API списка карт; HCE-F `02FE` запрещает эмуляцию физических карт.
- **Alternatives considered:** авто-синк карт из AquaDX (отклонён — нет API + нельзя эмулировать `012E`); только username-профиль (недостаточно).
- **Why chosen:** даёт максимум реально достижимой ценности без ложных обещаний; минимальные правки.
- **Consequences:** профиль начнёт работать; «синхронизация» = импорт/каталог + профиль, а не зеркало аккаунта.
- **Follow-ups:** если у AquaDX появится приватный эндпоинт списка карт (через токен/WebView-логин) — добавить настоящий синк отдельной фичей.
